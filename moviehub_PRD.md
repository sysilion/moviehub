# [PRD] moviehub: 롯데시네마 굿즈 및 이벤트 트래킹 시스템

## 1. 프로젝트 개요
**moviehub**는 롯데시네마의 이벤트 정보와 영화 관련 굿즈(아트카드, 키링 등)의 실시간 재고 현황을 자동으로 수집, 저장 및 관리하는 시스템입니다.

## 2. 목적 및 비전
- **데이터 통합:** 산재된 이벤트 및 굿즈 정보를 한곳에 모아 관리.
- **실시간성 확보:** 인기 굿즈의 소진 상태를 워킹 타임에 집중적으로 모니터링하여 정확한 정보 제공.
- **확장성:** SQLite로 시작하여 추후 대규모 트래픽 처리를 위한 PostgreSQL 마이그레이션이 용이한 구조 설계.

## 3. 핵심 기능 (Core Features)

### 3.1. 이벤트 및 굿즈 데이터 수집
- **이벤트 정보 완전 저장:** API 응답으로 오는 모든 필드(`EventID`, `EventName`, `ProgressStartDate`, `ProgressEndDate`, `ImageUrl`, `EventTypeName` 등)를 누락 없이 DB에 저장합니다.
- **키워드 기반 굿즈 탐색:** 이벤트 명칭에서 `증정`, `뱃지`, `아트카드`, `키링` 등의 키워드를 분석하여 굿즈 증정 가능성이 높은 이벤트를 식별합니다.
- **재고 수량 강제 조회:** 굿즈 데이터가 명시되지 않은 이벤트라도 키워드 조건 충족 시 재고 조회 API를 강제 호출합니다.
- **GiftID 스캔 로직:** 유효한 `GiftID`가 확인되지 않을 경우, 기존 ID 주변부(비어있는 번호 및 마지막 번호 이후 +10개)를 자동으로 스캔하여 잔여 수량을 확인합니다.

### 3.2. 지능형 스케줄링 시스템
서버 자원 효율화 및 정보의 정확도를 위해 시간대별로 조회 주기를 다르게 설정합니다.
- **워킹 타임 (09:00 ~ 18:00):** 굿즈 소진이 활발한 시간대로, **10분 ~ 15분 간격**으로 1회 조회를 수행합니다.
- **비업무 시간 (18:00 ~ 익일 09:00):** 소진 속도가 느린 시간대로, **1시간 ~ 3시간 간격**으로 1회 조회를 수행합니다.

### 3.3. 데이터베이스 설계
- **초기 단계:** 경량화를 위해 **SQLite**를 사용합니다.
- **설계 원칙:** SQLAlchemy 등의 ORM을 사용하여 추후 **PostgreSQL**로 환경 변수 설정만으로 마이그레이션이 가능하도록 추상화된 DB 레이어를 구축합니다.

## 4. 상세 요구사항 (Technical Requirements)

### 4.1. 데이터 저장 항목
- **Event Table:** ID, 이름, 구분 코드, 유형, 시작/종료일, 이미지 URL, 시네마 정보 등 전체 JSON 필드.
- **Inventory Table:** 이벤트 ID, 굿즈 ID, 영화관 ID, 지역명, 영화관명, 수량(Cnt), 최종 업데이트 일시.

## 5. 단계별 로드맵
1.  **Phase 1:** API 연동 모듈 및 SQLite 스키마 설계.
2.  **Phase 2:** 시간대별 차등 스케줄러 및 데이터 수집 엔진 구현.
3.  **Phase 3:** 굿즈 ID 자동 스캔 알고리즘 및 예외 처리 강화.
4.  **Phase 4:** PostgreSQL 마이그레이션 테스트 및 관리 도구 개발.
